# Session 2 — facilitator notes

Front-facing material for participants is in the session root `README.md`. This folder holds the facilitator-only guidance.

## Recap (open the session with this)

Whether participants continue from their own Session 1 output or pick up the premade reference versions, open with a recap that walks through the reference summaries:

- which prompt style produced them and why (for example, a directed prompt focused on reliability, frequency, and trust, because those recurred across sources);
- what the central tension is (reliability vs. personalization — see `../../session-1-research-synthesis/facilitator/premade-design-notes.md`);
- why the summaries are solid enough to build on.

For returning participants the recap is a model for justifying their own work; for drop-ins it's the context they need to use the premade versions.

## Why personas are required here

Personas aren't a deliverable for their own sake. Session 3 runs the UX review in character as these personas, so they're a working input the next session depends on. That's the reason they must be evidence-tied and hypothesis-labelled — make that explicit to the room.
